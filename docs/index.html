Features

Incremental by Hash

Only changed files are uploaded. A local manifest tracks SHA‑256 checksums to skip unchanged files.

Integrity Verification

Use --verify to re‑download after upload and confirm a matching SHA‑256. Trust, but verify.

S3 Versioning

Enable bucket versioning to recover from accidental edits or deletes. Roll back with confidence.

How it works

Backup Flow

  • Hash each file (SHA‑256) and compare with previous manifest
  • Upload only new/changed files to S3 (or MinIO)
  • Attach file hash in object metadata
  • Optionally re‑download to verify integrity
  • Write/update manifest.json locally and upload timestamped copy

Restore Flow

  • Read the latest manifest (local or from bucket)
  • Download the latest version of each file
  • Optional integrity re‑check on restore

Quick Start

Install

python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
aws configure  # set keys & region

Create & version your bucket

aws s3api create-bucket --bucket my-backups --region us-east-1
aws s3api put-bucket-versioning --bucket my-backups --versioning-configuration Status=Enabled

Run a backup

python backup.py backup \
  --source C:\Users\shri\Documents \
  --bucket my-backups \
  --prefix laptop-001 \
  --verify

Restore files

python backup.py restore \
  --bucket my-backups \
  --prefix laptop-001 \
  --restore-target .\restore \
  --verify

Why this matters

Reliability

Versioning + integrity checks reduce the chance of silent data loss and make rollbacks straightforward.

Efficiency

Incremental uploads save bandwidth and time; --dry-run shows changes before you commit.

SRE‑Ready

Demonstrates automation, observability mindset, and recovery planning — core SRE principles.

FAQ

Does it support MinIO?

Yes. Run MinIO locally and pass --endpoint http://127.0.0.1:9000.

What about encryption?

You can enable SSE‑S3/SSE‑KMS by extending the put_object call; see README roadmap.

How do I schedule backups?

Use Windows Task Scheduler or cron. Trigger the same CLI with your preferred cadence.

Large files?

Boto3 supports multipart uploads; add parallelism if you expect multi‑GB files.
